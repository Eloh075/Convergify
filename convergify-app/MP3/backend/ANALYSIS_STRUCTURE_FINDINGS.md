# Analysis Results Structure - Current Implementation

## ✅ ALL REQUIRED FIELDS ARE PRESENT

The endpoint `/api/analyses/{id}/results` currently returns **ALL** required fields for the dashboard redesign.

## Top-Level Fields ✓

- ✅ `overall_match_score` - Present (rounded to 1 decimal)
- ✅ `skill_matches` - Present (array with all required fields)
- ✅ `skill_gaps` - Present (array with all required fields)
- ✅ `recommendations` - Present (array)
- ✅ `market_insights` - Present (object with top_skills, competition_level, etc.)
- ✅ `career_paths` - Present (array)
- ✅ `job_details` / `analyzed_jobs` - Present (array with match scores per job)
- ✅ `narrative_report` - Present (markdown summary)
- ✅ `summary` - Present (total_skills_identified, matching_skills, missing_skills, jobs_analyzed)
- ✅ `optimized_resume` - Present (id, changes_count, validation_passed)
- ✅ `cache_stats` - Present (cached_jobs, tokens_saved, cache_hit_rate)
- ✅ **`match_breakdown`** - Present (must_have, nice_to_have, preferred with matched/total counts)
- ✅ **`top_strengths`** - Present (top 5 skills by relevance_score)
- ✅ **`top_gaps`** - Present (top 5 gaps by importance)

## Skill Matches Fields ✓

Each skill match includes:
- ✅ `skill` - Skill name
- ✅ `relevance_score` - Confidence (0-1)
- ✅ `resume_mentions` - Count of evidence snippets
- ✅ `job_mentions` - Count from market frequencies
- ✅ `match_type` - Exact/Semantic/Substitution/None
- ✅ `evidence` - Object with text_snippets, origin_locations, timeline, confidence
- ✅ **`canonical_group`** - Skill category/group (from classification)
- ✅ **`match_percentage`** - Relevance score as percentage (0-100)
- ✅ **`confidence_score`** - Same as relevance_score (0-1)
- ✅ **`importance_level`** - Must-have/Nice-to-have/Preferred
- ✅ **`market_demand`** - Market frequency (0-1)
- ✅ **`category`** - Same as canonical_group

## Skill Gaps Fields ✓

Each skill gap includes:
- ✅ `skill` - Skill name
- ✅ `importance` - Importance score (0-1)
- ✅ `market_demand` - Market frequency (0-1)
- ✅ `category` - Importance level (Must-have/Nice-to-have/Preferred)
- ✅ `learning_resources` - Array (currently empty, can be populated)
- ✅ **`importance_level`** - Must-have/Nice-to-have/Preferred
- ✅ **`why_matters`** - Explanation of why this skill is important
- ✅ **`jobs_requiring`** - Array of job titles requiring this skill

## Job Details Fields ✓

Each job in `job_details` / `analyzed_jobs` includes:
- ✅ `job_id` - Job UUID
- ✅ `title` - Job title
- ✅ `company` - Company name
- ✅ `location` - Job location
- ✅ `required_skills` - Array of skill names
- ✅ **`match_score`** - Individual job match percentage (0-100)

## Market Insights Fields ✓

- ✅ `total_jobs_analyzed` - Count of jobs
- ✅ `top_skills` - Array of skills with demand_score, job_count, importance
- ✅ `competition_level` - high/medium/low based on match score
- ✅ `average_salary` - Object with min, max, currency (placeholder data)
- ✅ `must_have_skills` - Top 5 must-have skills
- ✅ `nice_to_have_skills` - Top 5 nice-to-have skills
